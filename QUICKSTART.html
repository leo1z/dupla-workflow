Para nuevos usuarios

Empezar con Dupla Workflow

Guía paso a paso para configurar tu sistema en 15 minutos y tener el primer proyecto en marcha.

Requisitos

Antes de empezar: elige tu IDE

Dupla Workflow funciona en 3 IDEs. Elige UNO e instálalo primero.

⚡
VS Code + Claude Code
Instala: VS Code, luego extensión "Claude Code" en marketplace. Mejor para: desarrolladores que ya usan VS Code.
🔀
Antigravity
Instala: Antigravity (Google's agent IDE). Mejor para: cambiar entre Claude y Gemini sin cerrar la app.
💻
Claude Desktop
Instala: Claude.ai desktop app (Anthropic official). Mejor para: algo simple y directo.
💡 Si no sabes cuál elegir
Recomendado: VS Code + Claude Code. Es lo más usado. O Antigravity si quieres cambiar entre Claude y Gemini en la misma sesión.
Paso 1 de 5

Instalar Dupla Workflow (terminal)

Una sola vez. Abre tu terminal/command line y ejecuta estos comandos:

$ git clone https://github.com/leo1z/dupla-workflow.git $ cd dupla-workflow $ bash bin/install.sh
¿Qué hace bin/install.sh?
Detecta: ¿tienes VS Code? ¿Antigravity? ¿Claude Desktop?
Instala: 9 comandos (skills) en tu IDE (~/.claude/skills/ o ~/.agent/skills/).
Configura: automaciones (hooks) y versión del sistema.
Resultado: El sistema está listo. Después no tocas más la terminal hasta /new-project.
Paso 2 de 5

Configurar tu máquina (IDE)

Una sola vez, en tu IDE. Si ya tienes CLAUDE.md o SYSTEM.md, el sistema lo detecta y pregunta si actualizar.

1
Abre tu IDE
VS Code + Claude Code, Antigravity, o Claude Desktop. En cualquier carpeta (puede ser tu Downloads). Si en Antigravity, puedes elegir Claude o Gemini aquí — cualquiera funciona.
2
Escribe el comando (IDE input)
En el input de chat (abajo), escribe:
/setup-dupla
3
Responde preguntas (si las hay)
Si es primera vez: Preguntas agrupadas (quién eres, stack, proyectos, plugins). Responde todo de una vez.
Si ya existe CLAUDE.md/SYSTEM.md: El sistema pregunta "Encontré CLAUDE.md, ¿actualizar? [s/n]". Elige sí o no para cada archivo.
4
Confirmación
El sistema genera o actualiza: ~/.claude/CLAUDE.md (identidad), ~/.claude/SYSTEM.md (stack + proyectos), ~/.claude/PROBLEMS_GLOBAL.md (aprendizajes). Listo.
✅ Listo — tu máquina ya conoce quién eres
Los archivos generados se leen automáticamente en cada proyecto. Puedes editarlos si algo cambia (stack nuevo, proyecto nuevo, etc.).
Importante

Multi-IDE y cambiar de modelo (Claude ↔ Gemini)

Funciona en Antigravity, Claude Desktop, VS Code. Puedes cambiar de LLM entre sesiones.

3 IDEs soportados
Antigravity: Claude + Gemini en el mismo lugar. Claude Desktop: app nativa de Anthropic. VS Code: extensión Claude Code. Elige uno. El sistema funciona igual en los tres.
Model Routing
Usa Claude para: implementar, hacer cambios de arquitectura, debug. Usa Gemini para: research, análisis, segunda opinión. /checkpoint te dirá cuál modelo es mejor para lo siguiente.
💡 Flujo típico
Idea nueva → Gemini (/new-project + research) → Claude (roadmap + arquitectura) → Cualquiera (implementación) → /checkpoint (qué hacer después).
Paso 3 de 5

Crear tu primer proyecto

Dos opciones: proyecto nuevo desde cero, o integrar uno existente. Dupla Workflow se adapta a dónde estés.

Proyecto nuevo
Idea fresca. /new-project te pregunta: ¿problema, usuario, MVP, riesgos? Genera docs + carpetas automáticamente. Puedes empezar desde cualquier punto de madurez (idea nueva, ya investigada, MVP definido).
Proyecto existente
Ya tienes código, docs, historia. /adapt-project lee tu estructura actual, mapea docs viejos, genera lo que falta. Nada se borra, todo se integra.
1
Crea una carpeta nueva
En tu terminal:
mkdir mi-proyecto && cd mi-proyecto && git init
2
Abre Claude Code en esa carpeta
Navega a mi-proyecto/ en Claude Code.
3
¿De dónde estás partiendo?
Idea nueva: cuéntale todo (problema, usuarios, MVP, riesgos, restricciones). Ya investigada: menciona dónde está tu research. MVP definido: comparte dónde está tu roadmap. /new-project se adapta y salta pasos que ya hiciste.
4
Corre el comando de inicialización
Una vez que ya explicaste el proyecto:
/new-project
5
Revisa lo que se generó
Dupla Workflow crea automáticamente: CLAUDE.md (contexto del proyecto), docs/PROJECT_STATE.md (estado actual), docs/ROADMAP.md (plan), docs/ARCHITECTURE.md (decisiones técnicas).
✅ Tu proyecto está inicializado
Ahora tienes una estructura clara. En cada sesión, el sistema sabe dónde estás sin que tengas que re-explicar.
Paso 4 de 5

Primera sesión de trabajo

Cada vez que abras Claude Code en tu proyecto, este es el flujo.

1
Iniciar sesión
Escribe:
/new-session
Claude leerá automáticamente dónde quedaste. Te dirá: qué hacer, con cuál LLM trabajar, qué está bloqueando.
2
Trabajar
Haz lo que el sistema te indicó. Edita archivos, ejecuta comandos, resuelve problemas.
3
Guardar progreso (/checkpoint)
Antes de cerrar la sesión o cambiar de modelo:
/checkpoint
Qué hace:
1. Quick save (mid-sesión): "¿Qué hiciste?" → actualiza PROJECT_STATE.md
2. Close (end of day): idem + pregunta por resolver problemas → archiva en PROBLEMS.md
3. Handoff (cambiar LLM): idem + genera bloque de traspaso + propone push

En todos los casos: propone mensaje de commit + recomienda próximo modelo.
4
Hacer commit y push
Git push. Listo. La próxima sesión sabrá exactamente dónde quedaste.
✅ Sesión guardada
PROJECT_STATE.md es tu "memoria": qué se hizo, qué sigue, qué está bloqueando. Sin esto, cada sesión sería como empezar de cero.
Paso 5 de 5

Mantener el sistema actualizado

Ocasionalmente, revisa que todo esté coherente. Es rápido.

Cada 2-4 semanas
Corre /health-check para verificar que todos los docs del proyecto estén actualizados y el stack coincida con el código.
Mantenimiento
Cuando cambia algo importante
Si cambias el stack, agregar un nuevo LLM, o tomas una decisión arquitectónica grande: /update-context. Actualiza CLAUDE.md automáticamente.
Según necesidad
Referencia

Los 4 archivos que todo proyecto tiene

Después de /new-project, cada proyecto se ve así:

📋
PROJECT_STATE.md
El más importante
Tu "fuente de verdad". Dice: dónde estás ahora, qué estás haciendo, qué sigue, qué te bloquea. Se actualiza con /checkpoint cada sesión. Lee esto primero cada mañana.
## Status Updated: 2026-04-21 Next: Implementar login con Google
🗺️
ROADMAP.md
Plan completo
Divide el proyecto en fases (2 semanas cada una). Cada fase tiene un GO/NO-GO: condiciones para continuar. Te dice si estás en camino o necesitas ajustar.
🏗️
ARCHITECTURE.md
Decisiones técnicas
Stack elegido, diagramas, decisiones de diseño. "¿Por qué usamos Supabase en lugar de Firebase?" Respuestas acá. Para cuando necesites refactorar o agregar un nuevo dev.
⚙️
CLAUDE.md
Contexto del proyecto
Lo que Claude necesita saber para trabajar en este proyecto: cómo el stack se conecta, dónde viven los archivos críticos, reglas especiales (no tocar la base de datos en esta etapa), etc.
Comprensión

¿Cómo funciona sin que yo tenga que pedirlo?

Cada comando de Dupla Workflow lee archivos automáticamente. No necesitas decir "lee PROJECT_STATE".

🤖
/new-session
Lee PROJECT_STATE.md + git log → te dice dónde estás sin preguntar.
💾
/checkpoint
Pregunta qué hiciste, actualiza PROJECT_STATE automáticamente, propone commit.
🔍
/health-check
Revisa que PROJECT_STATE no esté desactualizado. Si lo está, avisa.
🆕
/new-project
Lee ROADMAP + ARCHITECTURE y crea PROJECT_STATE basado en eso.
Analogía
Dupla Workflow es como tener un asistente que sabe exactamente dónde archivas todo. Cuando dices "necesito trabajar en el login", el asistente sabe dónde está el código, qué se hizo antes, y qué sigue — sin que tengas que explicarlo cada vez.
Workflow

Individual vs Equipo

Mismo sistema, flujo ligeramente diferente.

👤
Individual
Flujo típico:
1. /new-session → Leo dónde quedé
2. Trabajo
3. /checkpoint → guardo estado
4. git push

Quién es el "dueño"? Tú. /checkpoint actualiza PROJECT_STATE.md y propone commit.

Cambiar de LLM: /checkpoint handoff → abre en Gemini → /new-session. El sistema sincroniza automáticamente.
👥
Equipo (2-4 personas)
Flujo típico:
1. /new-session (tu turno)
2. Trabajo en tu rama
3. /checkpoint close (final del día)
4. git push → merge a main
5. Siguiente dev: git pull → /new-session

Quién es el "dueño"? El equipo. PROJECT_STATE.md es compartido en main.

TEAM section: PROJECT_STATE.md tiene campo "Owner" + "Last handoff". Cada dev ve quién está de turno.
💡 Clave diferencia
Individual: guardas en tu rama local. Equipo: mergeas a main, así el siguiente dev ve el estado actualizado. El sistema no necesita comandos nuevos — solo /checkpoint close hace que el estado sea compartido.

Integra tu código actual sin perder nada. /adapt-project lo hace automático.

1
Ve a tu carpeta de proyecto
En tu terminal: cd mi-proyecto-existente
2
Corre el comando de adaptación
En Claude Code (abierto en esa carpeta):
/adapt-project
3
Revisa qué encuentra
Te mostrará: archivos actuales, docs que faltan, cómo mapear docs viejos a la nueva estructura. Todo se preserva, nada se borra.
4
Aprueba la integración

✅ Tu proyecto existente está integrado
Ahora tienes PROJECT_STATE.md, ROADMAP.md, y el sistema sabe dónde estás en el código.
FAQ

Preguntas frecuentes

¿Y si tengo un proyecto que ya existe?
Corre /adapt-project en lugar de /new-project. El sistema detecta qué docs faltan y genera solo lo necesario — sin pisar lo que ya existe.
¿Cuánto se tarda en /new-project?
2-3 minutos máximo. El sistema genera 4 archivos. Luego puedes editarlos si quieres ser más específico.
¿Puedo usar Gemini en lugar de Claude?
Sí. El sistema tiene modelo-routing: defines quién hace qué. MODEL_ROUTING.md te dice cuándo usar Gemini (investigación, code review) y cuándo usar Claude (código, arquitectura).
¿Qué pasa si no corro /checkpoint?
La próxima sesión no sabrá qué pasó. Dupla Workflow puede reconstruir desde git log, pero es más lento. /checkpoint toma 30 segundos — úsalo.
¿Los archivos se pushen a Git?
Sí, PROJECT_STATE.md, ROADMAP.md, y otros se pushean. Así tu equipo (o tu yo del futuro) ve el estado. CREDENTIALS.md nunca se pushea — está en .gitignore.
¿Qué es MCP?
Opcional. File system access para Claude (solo en Claude Desktop/Code). /setup-dupla te pregunta si quieres habilitarlo. Si no sabes qué es, elige "no". No lo necesitas para usar Dupla Workflow.
¿Puedo cambiar de Gemini a Claude durante un proyecto?
Sí, en Antigravity. /checkpoint te recomienda cuál modelo es mejor para lo siguiente. Si quieres cambiar: /checkpoint, cambia de LLM en Antigravity, luego /new-session en el nuevo modelo.
Mi idea ya está investigada, ¿qué hago?
/new-project lo detecta en Phase 0. Responde cuál es tu estado (idea nueva, investigada, MVP definido). El sistema salta pasos que ya completaste.
¿Funciona en equipo?
Sí. PROJECT_STATE.md se pushea, así todos ven dónde está el proyecto. CLAUDE.md del proyecto tiene reglas compartidas. Si hay conflictos, /health-check los detecta.
